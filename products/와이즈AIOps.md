# 와이즈AIOps (WiseAIOps)

> **분류:** AI 개발 플랫폼 (MLOps)  
> **버전:** Wise AIOps v1.0  
> **최종 수정일:** 2026-07-03

---

## 1. 제품 개요

데이터 전처리, 모델 개발·학습, 평가, 모니터링, 배포까지  
**AI 개발 전 단계를 하나의 환경에서 지원**하는 엔드투엔드 AI 플랫폼.

플랫폼은 6개의 대분류 모듈로 구성된다:
`Data Manager` · `Workflow` · `Monitoring` · `Resource` · `User Manager` · `Model Manager`

---

## 2. 주요 기능

> 표기 규칙: `### 대분류` → `#### 중분류` → `- **소분류**: 상세 설명`  
> 소분류 아래 `·`로 나열된 항목은 해당 기능이 제공하는 세부 옵션(컴퍼넌트·알고리즘 등)이다.

### 2.1 Data Manager

데이터 수집·저장·조회·스케줄링 등 데이터 라이프사이클 관리 모듈.

#### 데이터셋 관리
- **데이터셋 추가**: 연결관리에 등록된 저장소에서 테이블 또는 파일을 통해 데이터셋을 생성하는 기능
- **데이터셋 목록**: 등록된 데이터셋의 리스트 및 정보를 확인하는 기능
- **데이터셋 삭제**: 데이터셋을 목록에서 삭제하는 기능
- **데이터셋 다운로드**: 데이터셋을 다운로드하는 기능
- **데이터셋 API 공유**: 데이터셋을 API로 공유하는 기능
- **데이터셋 권한 공유**: 데이터셋을 다른 사용자에게 공유하는 기능
- **데이터셋 확인**: 데이터셋 클릭 시 통계량 및 마우스 드래그를 통해 데이터를 확인하는 기능

#### 연결 관리
- **연결 추가**: 외부 저장소를 등록하여 활용할 수 있도록 하는 기능

#### 하이브쿼리
- **테이블 및 컬럼 확인**: 왼쪽 사이드 트리 구조에서 테이블 목록 및 테이블의 컬럼을 확인하는 기능
- **쿼리 실행**: SELECT 쿼리를 실행하고 데이터를 확인하는 기능

#### 스토리지 뷰어
- **폴더 생성**: AIOps 기본 파일저장소에 폴더를 생성하는 기능
- **파일 생성**: 데이터베이스 또는 파일 업로드로 AIOps 기본 파일저장소에 파일을 생성하는 기능
- **폴더 기능**: 선택한 폴더에 대한 이름변경/이동/삭제 기능
- **파일 기능**: 선택한 파일에 대한 이름변경/이동/복사/삭제 기능

#### 스케줄 관리
- **스케줄 등록**: 생성한 워크플로우를 스케줄에 등록하여 자동으로 실행할 수 있도록 하는 기능
- **스케줄 로그 확인**: 목록에서 스케줄을 클릭 시 해당 스케줄에 대한 로그를 확인하는 기능
- **스케줄 정지/시작**: 스케줄을 정지하거나 시작하는 기능
- **스케줄 수정**: 등록한 스케줄의 시간·워크플로우·이름 등을 수정하는 기능
- **워크플로우 확인**: 해당 스케줄의 워크플로우가 어떤 형태인지 확인하는 기능
- **스케줄 삭제**: 등록한 스케줄을 삭제하는 기능

### 2.2 Workflow

컴퍼넌트를 드래그·연결하여 데이터 처리부터 모델 학습까지 수행하는 저코드 파이프라인 모듈.

#### 정형 데이터
- **입력 컴퍼넌트**: 컴퍼넌트를 통해 데이터를 불러오는 기능
  - 지원 소스: 스토리지(AIOps 기본 스토리지) · 하이브 · 데이터베이스 · 데이터셋(AIOps 관리 데이터셋) · SFTP/FTP · API
- **워크플로우 컴퍼넌트**: 저장된 워크플로우를 다른 워크플로우에서 재사용하는 기능
- **변환 컴퍼넌트**: 정형 데이터를 가공·변환하는 기능
  - 지원 변환: 타입 · 병합 · 정렬 · 조인 · 분할 · 슬라이스 · 파생열 · 조건부 파생열 · 집계 · 비율 샘플링 · 결측값 처리 · 시계열 · 이상값 처리 · 열 선택 · 그룹화 · 파이썬 코드 · 컬럼명 변경 · 변수중요도
- **출력 컴퍼넌트**: 컴퍼넌트를 통해 데이터를 저장하는 기능
  - 지원 대상: 데이터베이스 · 데이터셋(AIOps 관리 데이터셋)

#### 비정형 데이터
- **이미지 입력 컴퍼넌트**: 컴퍼넌트를 통해 이미지 데이터를 불러오는 기능
  - 지원 소스: 스토리지(AIOps 기본 스토리지) · 데이터셋(AIOps 관리 데이터셋)
- **이미지 변환 컴퍼넌트**: 컴퍼넌트를 통해 이미지 데이터에 대해 라벨링 및 변환을 하는 기능
  - 지원: 이미지 라벨
- **이미지 출력 컴퍼넌트**: 컴퍼넌트를 통해 이미지 데이터를 저장하는 기능
  - 지원 대상: 데이터셋(AIOps 관리 데이터셋)

#### 모델
- **분류 모델**: 컴퍼넌트를 통해 정형 데이터에 대해 분류 모델을 학습·저장하는 기능
  - 알고리즘: Decision Tree · Gradient Boosting · Random Forest · XGBoost · Logistic Regression · Ensemble
- **회귀 모델**: 컴퍼넌트를 통해 정형 데이터에 대해 회귀 모델을 학습·저장하는 기능
  - 알고리즘: Decision Tree · Gradient Boosting · Random Forest · XGBoost · ElasticNet · Ensemble
- **군집 모델**: 컴퍼넌트를 통해 정형 데이터에 대해 군집 모델을 학습·저장하는 기능
  - 알고리즘: K-means · Gaussian Mixture · K-Medoids
- **언어 모델**: 텍스트 칼럼이 있는 정형 데이터를 프롬프트에 바인딩하여 언어 모델을 학습·저장하는 기능
  - 모델: LLAMA3
- **이미지 모델**: 이미지 데이터에 대해 오브젝트 디텍션 모델을 학습·저장하는 기능
  - 모델: YOLO
- **커스텀 모델**: 사용자가 Model Manager의 커스텀 모델 관리에서 등록한 외부 모델을 사용하여 학습·저장하는 기능
- **모델 관리**: 모델 간 성능 비교/필터링 및 프롬프트를 통해 언어 모델을 학습하는 기능
  - 세부: 모델 비교 · 모델 필터링 · 동적 프롬프트
- **활용**: 저장된 모델을 불러와서 활용하는 기능
  - 세부: 모델 예측 · 모델 배포

#### 공통
- **워크플로우 제작**: 컴퍼넌트를 드래그하고 연결하여 워크플로우를 만드는 기능
- **워크플로우 실행**: 사용자가 만든 워크플로우를 실행하고 실행 시간 등 결과를 확인하는 기능
- **워크플로우 저장**: 워크플로우 컴퍼넌트에서 재활용하거나 스케줄에 설정할 수 있도록 워크플로우를 저장하는 기능
- **워크플로우 관리**: 워크플로우 불러오기/이름변경/삭제/공유 관리 기능
- **데이터 스키마**: 컴퍼넌트별로 데이터 스키마가 어떻게 변화하는지 Before/Current로 확인하는 기능

### 2.3 Monitoring

AIOps에서 실행되는 모든 프로세스를 실시간으로 관리하는 모듈.

#### AIOps 프로세스의 실시간 관리
- **자원 사용량**: 모든 프로세스에 대해 실시간 자원 사용량(CPU·Memory·GPU)을 라인차트로 확인하는 기능
- **프로세스 관리**: 모든 프로세스에 대해 실시간 정보 확인 및 Kill(강제 종료) 관리 기능

### 2.4 Resource

쿠버네티스 클러스터와 플랫폼 자원을 관리하는 모듈.

#### 클러스터 관리
- **클러스터 추가**: AIOps에서 관리할 쿠버네티스 클러스터를 등록하는 기능
- **클러스터 목록**: 등록된 클러스터 목록을 확인하는 기능
- **클러스터 수정**: 등록된 클러스터를 수정하는 기능
- **클러스터 삭제**: 등록된 클러스터를 삭제하는 기능
- **클러스터 구성 확인**: 트리 구조를 통해 클러스터의 정상 작동 여부와 구성도를 확인하는 기능
- **클러스터 상세 정보 확인**: 트리에서 우클릭하여 설명/YAML 보기로 특정 구성원의 상세 정보를 확인하는 기능
- **클러스터 실시간 모니터링**: 트리에서 클러스터·노드를 우클릭하여 실시간 노드/파드 가동률·디스크 사용률·CPU/Memory/GPU 사용률을 확인하는 기능

#### 플랫폼 관리
- **AIOps 플랫폼 구성도**: AIOps 전체 플랫폼의 구성도와 상태를 확인하는 기능
- **어플리케이션 정보**: 어플리케이션 컴퍼넌트를 클릭하여 상태 및 구성 설정값을 확인하는 기능
- **실시간 로그 확인**: 어플리케이션 컴퍼넌트를 우클릭하여 실시간 로그를 확인하는 기능
- **모델 배포 Pod 생성**: 노드 컴퍼넌트를 우클릭하여 저장된 모델에 자원을 할당해 배포·활용하는 기능
- **Pod 삭제**: Pod를 삭제하는 기능

### 2.5 User Manager

사용자·그룹 및 접속 이력을 관리하는 모듈.

#### 사용자 관리
- **그룹 추가**: AIOps 사용자 그룹을 생성하는 기능
- **사용자 추가**: AIOps 사용자를 추가하는 기능
- **그룹/사용자 구성도 확인**: 트리 구조로 그룹 목록 및 각 그룹의 사용자를 확인하는 기능
- **그룹 수정**: 트리에서 그룹을 선택하여 정보를 수정하는 기능
- **그룹 삭제**: 트리에서 그룹을 선택하여 삭제하는 기능
- **사용자 수정**: 트리에서 사용자를 선택하여 정보를 수정하는 기능
- **사용자 삭제**: 트리에서 사용자를 선택하여 삭제하는 기능

#### 사용자 접속이력
- **접속이력 확인**: 사용자들의 로그인/접속/로그아웃을 확인하는 기능

### 2.6 Model Manager

학습된 모델의 이력·커스텀 모델·프롬프트를 관리하는 모듈.

#### 모델 이력
- **모델 목록 확인**: 사용자들이 워크플로우로 학습·저장한 모델 목록을 확인하는 기능
- **모델 삭제**: 해당 모델을 목록에서 삭제하는 기능
- **모델 다운로드**: 해당 모델을 압축파일로 다운로드하는 기능
- **모델 상세정보**: 모델의 파라미터/평가지표/변수영향도/예측차트 등 상세 정보를 확인하는 기능
- **모델 알림**: 해당 모델의 예측 · 배포 · **재학습 필요 여부**에 대한 모델 사용 로그를 확인하는 기능
- **모델 이력 확인**: 버전별 파라미터/성능 비교, 특정 버전 모델 정보, 모델이 만들어진 워크플로우 정보를 확인하는 기능

#### 커스텀 모델 관리
- **커스텀 모델 추가**: 사용자가 AIOps가 아닌 외부에서 만든 파이토치(PyTorch) 모델을 등록하는 기능
- **커스텀 모델 목록**: 사용자가 등록한 커스텀 모델 목록을 확인하는 기능
- **커스텀 모델 삭제**: 해당 커스텀 모델을 삭제하는 기능
- **커스텀 모델 상세정보**: 커스텀 모델의 가중치 파라미터, 레이어 등 상세 정보를 확인하는 기능

#### 프롬프트 관리
- **프롬프트 추가**: 새로운 프롬프트를 추가하는 기능
- **프롬프트 목록**: 등록된 프롬프트 목록을 확인하는 기능
- **프롬프트 삭제**: 해당 프롬프트를 삭제하는 기능
- **프롬프트 상세정보**: 특정 프롬프트의 상세 정보를 확인하는 기능

### 2.7 지능형 운영 — 드리프트 감지 및 자동 재학습

> ⚠️ **확인 필요:** 이 절의 기능은 업로드된 v1.0 기능리스트 PDF에는 명시되어 있지 않으나, 제품 개요·제안 포인트상 제공(또는 로드맵)되는 기능으로 추가한 항목이다. 실제 지원 범위·명칭은 개발팀 확인 후 확정할 것. (관련 단서: Model Manager의 "모델 알림 → 재학습 필요 여부", Data Manager의 "스케줄 관리")

#### 드리프트 감지
- **데이터 드리프트 감지**: 배포 이후 유입되는 입력 데이터의 분포가 학습 데이터 대비 변화(드리프트)했는지 감지하고 알림을 제공하는 기능
- **모델 드리프트 감지**: 배포된 모델의 예측 성능이 시간 경과에 따라 저하되는지(성능 드리프트) 감지하고 재학습 필요 여부를 판단·알림하는 기능

#### 이벤트 기반 자동 재학습
- **이벤트 기반 모델 재실행**: 드리프트 감지·성능 임계치 미달·신규 데이터 유입 등 특정 이벤트를 트리거로 학습 워크플로우를 자동 재실행하는 기능 (스케줄 기반 주기 실행과 대비되는 이벤트 기반 실행)
- **재학습 파이프라인 자동화**: 재학습 → 평가 → (조건 충족 시) 자동 배포까지 이어지는 파이프라인을 자동화하는 기능

## 3. 도입 효과

- AI 개발부터 운영까지 일원화된 관리 환경 제공
- 모델 재현성 보장 및 버전 추적
- 배포 후 드리프트 감지·자동 재학습으로 모델 성능 지속 유지
- 망분리 환경 온프레미스 구축 지원

---

## 4. 제안서 작성 시 강조 포인트

- **엔드투엔드 AI 플랫폼** — 단계별 툴 난립 없이 단일 환경에서 전 과정 관리
- **공공기관 망분리 환경 지원** — 보안 요구사항 충족
- **AI 모델 거버넌스** — 모델 버전·이력·성능 추적으로 설명 가능한 AI 구현
- **지능형 운영(MLOps)** — 데이터/모델 드리프트 감지 및 이벤트 기반 자동 재학습으로 무중단 모델 품질 관리
- **WiseProphet 연계** — 비전문가도 활용 가능한 저코드 AI 개발 환경 통합 제공
